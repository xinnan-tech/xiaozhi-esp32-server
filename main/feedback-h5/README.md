# 客户反馈系统 - 使用指南

## 系统架构
- **MySQL**: 3张表（feedback_store, feedback_employee, feedback_record）
- **Java manager-api** (8002): 门店/员工/记录 CRUD（注意：Java代码需重建镜像才生效，当前H5使用mock数据降级）
- **Python xiaozhi-server** (18000/18003): WebSocket语音对话 + `/feedback/process` AI处理接口
- **H5前端** (8007): 客户使用页面，含Live2D数字人

## 完整测试流程

### 第一步：访问H5页面
浏览器打开：`http://127.0.0.1:8007/#/home?code=558744`

应该看到：
- 自动识别门店"康悦养生店"
- 展示3名员工卡片（1号张三-店长、2号李四-优秀、3号王五-实习）

### 第二步：选择员工 → 开始反馈
点击任意员工 → 点击"开始语音反馈" → 跳转到语音对话页

### 第三步：智能体绑定（首次）
H5 会显示**6位验证码**（如：544341）

**管理员操作**：
1. 打开智控台 `http://localhost:8002`
2. 进入"智能体管理" → 新增智能体
3. 输入6位验证码 → 绑定设备
4. 配置该智能体的：
   - LLM（如 DeepSeek）
   - ASR（如 FunASR）
   - TTS（如 EdgeTTS）
   - 系统提示词（7个固定问题引导逻辑，见下方）

### 第四步：再次拨号 → 开始对话
绑定后再次访问H5 → 自动连接 → Live2D数字人开始按7个问题提问

### 第五步：对话结束 → AI 处理
对话结束（WebSocket断开）后，H5 自动调用 `/feedback/process`：
- 第1步：ASR 文本规整
- 第2步：QA 结构化
- 第3步：生成美团/大众点评好评

### 第六步：复制好评 + 跳转
- 点击"复制好评" → 自动 TTS 播报"已复制"
- 点击"打开大众点评/美团" → 唤起APP或网页版

## 智能体提示词模板

在智控台创建智能体时填入：

```
你是一位客户服务回访助手，名叫小智。你的任务是按照顺序依次向客户提出以下7个问题，每次只问一个问题，等待客户回答后再问下一个：

1. 您来店里多久了？
2. 今天做的什么项目？
3. 您是几点过来做项目的？
4. 您来之前觉得哪里不舒服呢？
5. 本次服务解决您的问题了么？
6. 您对本次服务满意么？
7. 您对我们有什么好的建议么？

规则：
- 严格按顺序提问，不可跳过或打乱顺序
- 每次只问一个问题，等客户回答后再问下一个
- 客户回答后简单回应（如"好的，了解了"），然后自然过渡到下一个问题
- 如果客户没听清，重复当前问题
- 不要询问客户怎么称呼、手机号、手机尾号或手机号后四位，这些信息会在对话结束后的表单中填写
- 不要偏离这7个问题去聊其他话题
- 7个问题全部回答完毕后，感谢客户并说"您的反馈我们已记录，感谢您的参与！再见"
```

## 启动/停止

### 启动 H5
```bash
cd main/feedback-h5
python start.py
```

### 重启 Python 服务（修改 feedback_handler.py 后）
```bash
docker restart xiaozhi-esp32-server
```

### 查看 Python 日志
```bash
docker logs xiaozhi-esp32-server --tail 50
```
