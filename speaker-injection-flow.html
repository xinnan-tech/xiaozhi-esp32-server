Xiaozhi server · 方案 A 设计

speaker 注入:让小智接收 lebot 识别好的说话人

lebot 代理连接下,小智 server 接收外部已识别的 speaker,跳过自己的 VoiceprintProvider。 下游 ASR / 注入 / LLM 全部不碰;直连设备路径零改动。

范围 · 仅小智 server 侧 改动 · 1 新 handler + base.py ~8 行 + connection.py ~5 行 协议 · 普通 JSON 文本帧
角色 设备 ESP32/App lebot 小智 server
语义 新增 现有不变 ◆ 决策

端到端时序

图 1 / 三角色 · 时间自上而下

关键:speaker 帧在说话中途(攒满 ≈900ms)就发出,远早于小智消费它的时刻(handle_voice_stop,在 listen stop 之后)。

三种场景

speaker 帧 vs handle_voice_stop 先后
A正常 · ≥900ms
到达:说话中途
处理:早已就绪,零等待
占比:绝大多数
B网络/VPR 慢
到达:晚于 stop 几百 ms
处理:wait_for 0.3s 接住
结果:拿到 speaker
C短语音 <900ms
到达:永不(lebot 不发)
处理:超时 fallback
结果:「未知说话人」
