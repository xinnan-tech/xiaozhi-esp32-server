# 本地数据库格式与说明文档

> 数据库文件：`data/growth_service.db`
> 类型：SQLite（WAL 模式，开启并发读写优化）
> 生成依据：`src/database/models.py`（DDL）、`src/database/repository.py`（读写逻辑）、`src/services/profile_enricher.py`（13 维画像）、`src/utils/validators.py`（校验逻辑）
> 当前快照：7 张表（6 张业务表 + 1 张系统表 `sqlite_sequence`），10 个索引

---

## 一、总览

| # | 表名 | 用途 | 当前行数 | 唯一约束 / 主键 |
|---|------|------|----------|------------------|
| 1 | `profile_enrichment` | 儿童画像扩展字段（13 维画像 JSON） | 1 | `device_id` UNIQUE |
| 2 | `emotion_cache` | 每日情绪缓存（供周报复用） | 11 | `(device_id, cache_date)` |
| 3 | `capability_history` | 五维能力周度历史趋势 | 10 | `(device_id, week_start, dimension_key)` |
| 4 | `task_execution_log` | 任务执行日志（日摘要/周报） | 25 | — |
| 5 | `device_state` | 设备处理状态（增量水位线） | 4 | `device_id` PK |
| 6 | `config` | 运行时配置 KV | 2 | `key` PK |
| 7 | `sqlite_sequence` | SQLite 自增序列（系统表） | 4 | — |

数据清理策略（`repository.cleanup_old_data`）：
- `emotion_cache`：保留 14 天
- `task_execution_log`：保留 90 天
- `capability_history`：保留 26 周（半年趋势）

## 二、各表详细结构

### 2.1 `profile_enrichment`（画像扩展字段 ⭐重点）

存储儿童 12+1 维画像特征，由 `ProfileEnricher` 通过 LLM（qwen-max）从本周能力评估、兴趣、情绪、高光片段中提炼，每周更新一次。

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INTEGER | PK AUTOINCREMENT | — | 自增主键 |
| `device_id` | TEXT | NOT NULL UNIQUE | — | 设备 UUID（一台设备仅一条记录，覆盖更新） |
| `updated_at` | TEXT | NOT NULL | — | 本次更新时间，ISO 8601 |
| `update_source` | TEXT | — | `'weekly_report'` | 更新来源 |
| `update_confidence` | REAL | — | `0.85` | 本次更新置信度 |
| `extension_fields` | TEXT | NOT NULL | — | **完整画像 JSON（13 维 + 4 元数据）** |
| `previous_fields` | TEXT | — | NULL | 上一版本 JSON（用于对比，首次写入为 NULL） |
| `version` | INTEGER | — | `1` | 版本号，每次 UPSERT +1 |
| `create_time` | DATETIME | — | `CURRENT_TIMESTAMP` | 首次创建时间 |
| `update_time` | DATETIME | — | `CURRENT_TIMESTAMP` | 记录更新时间 |

索引：`idx_profile_device(device_id)`、`sqlite_autoindex_profile_enrichment_1`（device_id 唯一）

UPSERT 逻辑（`repository.upsert_profile`）：写入前读取旧版本，将旧 `extension_fields` 移入 `previous_fields`，`version` 自增。

### 2.2 `emotion_cache`（每日情绪缓存）

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INTEGER | PK AUTOINCREMENT | — | 自增主键 |
| `device_id` | TEXT | NOT NULL | — | 设备 UUID |
| `cache_date` | TEXT | NOT NULL | — | 日期 `YYYY-MM-DD` |
| `emotions` | TEXT | NOT NULL | — | 情绪分布 JSON，如 `{"happy":0.35,"curious":0.25,...}` |
| `dominant_emotion` | TEXT | — | NULL | 当日主导情绪 |
| `create_time` | DATETIME | — | `CURRENT_TIMESTAMP` | 创建时间 |

唯一约束：`UNIQUE(device_id, cache_date)`（同日同设备覆盖更新）
索引：`idx_emotion_device_date(device_id, cache_date)`

> 注意：实际数据中情绪 key 存在中英文混用（早期 `happy/curious/excited/calm/frustrated`，近期 `开心/好奇/兴奋/平静/沮丧`），后续接入需做归一。

### 2.3 `capability_history`（能力周度历史）

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INTEGER | PK AUTOINCREMENT | — | 自增主键 |
| `device_id` | TEXT | NOT NULL | — | 设备 UUID |
| `week_start` | TEXT | NOT NULL | — | 周一日期 `YYYY-MM-DD` |
| `dimension_key` | TEXT | NOT NULL | — | 维度 snake_case key（见下方枚举） |
| `total_score` | REAL | NOT NULL | — | 维度总分（满分通常 20） |
| `max_score` | REAL | NOT NULL | — | 维度满分值（固定 20） |
| `sub_scores` | TEXT | NOT NULL | — | 子维度评分 JSON：`[{key,score,evidence}]` |
| `age_at_week` | REAL | — | NULL | 儿童当周实际年龄（岁，如 6.0） |
| `create_time` | DATETIME | — | `CURRENT_TIMESTAMP` | 创建时间 |

唯一约束：`UNIQUE(device_id, week_start, dimension_key)`（同设备同周同维度覆盖）
索引：`idx_cap_history_device_week(device_id, week_start)`

`dimension_key` 枚举（5 维）：
- `language_expression`（语言表达力）
- `logical_thinking`（逻辑思维力）
- `social_understanding`（社交理解力）
- `knowledge_integration`（知识整合度）
- `creative_imagination`（创造想象力）

`sub_scores` JSON 中 `key` 对应子维度（每维度 4 项，详见 `src/models.py` 的 `SUB_DIMENSION_NAME_MAP`）。

## 三、扩展字段 `extension_fields` 详细说明 ⭐

`profile_enrichment.extension_fields` 存储一段完整 JSON，由 **13 个画像维度对象 + 4 个元数据字段** 组成，共 17 个顶层 key。

### 3.1 顶层结构

```
{
  // ── 13 个画像维度对象 ──
  "developmentalBaseline": {...},   // 1. 发展基线
  "interestProfile":        {...},   // 2. 兴趣偏好
  "emotionProfile":         {...},   // 3. 情绪基线
  "languageStyle":          {...},   // 4. 语言风格
  "capabilitySnapshot":     {...},   // 5. 能力快照
  "interactionHabits":      {...},   // 6. 互动习惯
  "growthSignals":          {...},   // 7. 成长信号
  "socialContext":          {...},   // 8. 社交关系
  "knowledgeProfile":       {...},   // 9. 知识图谱
  "safetyProfile":          {...},   // 10. 安全档案
  "parentIntent":           {...},   // 11. 家长意图
  "characterMatchIndex":    {...},   // 12. 角色匹配指数
  "temporalDynamics":       {...},   // 13. 时间动态

  // ── 4 个元数据字段（由 ProfileEnricher.extract 写入） ──
  "deviceId":          "019ec95e-...",
  "updatedAt":         "2026-06-18T13:13:56+00:00",
  "updateSource":      "weekly_report",
  "updateConfidence":  0.85
}
```

> 校验逻辑见 `validators.LLMOutputParser.validate_profile`：13 个维度 key 缺失或非 dict 时，自动填充为空对象 `{}`，保证结构完整。

### 3.2 各维度字段详解（基于实际样例数据）

#### 1) `developmentalBaseline` 发展基线
| 子字段 | 类型 | 说明 |
|--------|------|------|
| `ageRange` | string | 年龄段描述，如 "学龄前至小学低年级" |
| `languageStage` | string | 语言发展阶段 |
| `playLevel` | string | 游戏水平 |
| `emotionRegulation` | string | 情绪调节能力 |

#### 2) `interestProfile` 兴趣偏好
| 子字段 | 类型 | 说明 |
|--------|------|------|
| `currentTopics` | array | 当前话题列表，每项见下 |
| `trend` | string | 趋势描述 |
| `interactionMode` | string | 互动模式 |

`currentTopics[]` 元素：
| 字段 | 类型 | 说明 |
|------|------|------|
| `tag` | string | 话题标签，如 "身体健康" |

| `confidence` | float | 置信度（0-1） |
| `firstMentioned` | string | 首次提及日期 `YYYY-MM-DD`（由 `_sanitize_dates` 校正，早于 week_start 时被替换为 week_start） |

#### 3) `emotionProfile` 情绪基线
| 子字段 | 类型 | 说明 |
|--------|------|------|

#### 4) `languageStyle` 语言风格

#### 7) `growthSignals` 成长信号

#### 9) `knowledgeProfile` 知识图谱

| `cognitiveMisconceptions` | array | 认知误区 |

#### 12) `characterMatchIndex` 角色匹配指数

| 子字段 | 类型 | 说明 |
|--------|------|------|
| `profileStability` | string | 画像稳定性，如 "波动中" |
| `trendChange` | string | 趋势变化 |
| `seasonalPattern` | string | 季节性模式 |

### 3.3 元数据字段（4 个）

| 字段 | 类型 | 说明 |
|------|------|------|
| `deviceId` | string | 设备 UUID（与表 `device_id` 一致） |
| `updatedAt` | string | 本次更新时间 ISO 8601（与表 `updated_at` 一致） |
| `updateSource` | string | 固定 `"weekly_report"` |
| `updateConfidence` | float | 固定 `0.85` |

### 3.4 版本管理机制

- 每次 `ProfileEnricher.extract` 产出新画像后，调用 `repository.upsert_profile`
- 旧 `extension_fields` → 移入 `previous_fields`（保留上一版本用于对比）
- `version` 自增（当前样例为 version=2）
- 13 维度内容"与上周数据对比，有变化才更新，无变化保留原值"（见系统提示词）

## 四、能力维度相关常量（参考）
