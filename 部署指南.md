# 自构建部署指南（含自定义改动）

> 本文件用于：在新机器上 `git clone` 本仓库后，按此文档一键起服务。
> 对应 compose：仓库根 `docker-compose-selfbuild.yml`
> 与官方 `main/xiaozhi-server/docker-compose_all.yml` 的区别：server/web 一律**本地构建**（自定义代码烤进镜像），不依赖任何主机路径或 volume 挂载源码。

---

## 0. 这个部署包含什么自定义改动

- **aipet 记忆 provider**：智控台 Memory 下拉可见（FTS5 本地记忆）
- **qwen / 豆包 缓存开关**：LLM 模型编辑页显示「是否开启缓存」开关（boolean）
- **doubao_cache provider**：豆包 Responses API Session 缓存
- **server 端核心定制**：当天日程注入（`connection.py`）+ xiaozhi-memory FTS5 检索修复（分词 / 字符串相似度 / 整句天花板）
- 以上全部已提交进 git，`git clone` 即含。

---

## 1. 前置条件

- Docker + Docker Compose（compose v2 插件）
- **国内机器**：`/etc/docker/daemon.json` 配置 registry-mirrors（拉 Docker Hub 的 node / maven / jre / python）：

  ```json
  {
    "registry-mirrors": ["https://docker.1ms.run"]
  }
  ```
  配完 `sudo systemctl restart docker`。
- mysql / redis 走 `docker.1ms.run` 镜像（compose 里已写死，无需额外配置）。
- server 镜像的 base 走 `ghcr.nju.edu.cn`（NJU 镜像，compose/Dockerfile 里已写死）。

---

## 2. 首次部署：准备运行时文件

compose 用 volume 挂载运行时数据，**首次启动前必须准备**（这些不入镜像、不入 git）：

```bash
cd xiaozhi-esp32-server   # 仓库根

# (1) 配置目录：放 config.yaml（智控台 + 模型配置）
mkdir -p data
#     把你的 config.yaml 放到 data/config.yaml
#     （可从旧机 /home/aipet/coding/xiaozhi-server/data/config.yaml 拷过来）

# (2) ASR 模型（语音识别，按需；不放则语音相关功能不可用）
mkdir -p models/SenseVoiceSmall
#     把 SenseVoiceSmall/model.pt 放到 models/SenseVoiceSmall/model.pt
```

> `config.yaml` 里 LLM 选用的模型 type、API key 等决定运行行为；智控台（8002）也能在线改。

---

## 3. 一键启动

```bash
docker compose -f docker-compose-selfbuild.yml up -d --build
```

首次会构建两个镜像（耗时主要在拉 base + 装依赖，约 10~20 分钟）：
- `xiaozhi-esp32-server:server_aipet`（~10GB，含全部自定义 Python）
- `xiaozhi-esp32-server:web_aipet`（~570MB，含 aipet/缓存 changelog）

后续启动（代码没变）去掉 `--build` 即可秒起。

## 4. 验证

```bash
# 容器状态（应都是 Up）
docker compose -f docker-compose-selfbuild.yml ps

# 智控台（manager-web）
浏览器打开 http://<服务器IP>:8002

# ws 服务端口
8000  # 小智设备连接
8003  # 视觉分析 http 接口
```

**智控台验证**：
- 「Memory 类型」下拉应出现 **aipet**
- 编辑 qwen（openai 类型）/ 豆包模型，应看到「**是否开启缓存**」开关
- LLM「新增模型」下拉应出现「豆包(Session缓存)」

> provider 列表是**数据库驱动**的（`ai_model_provider` 表），web 容器启动时 Liquibase 自动跑 changelog 注册 aipet + 缓存开关，无需手动操作。

## 6. 更新代码后重新部署

```bash
git pull
docker compose -f docker-compose-selfbuild.yml up -d --build
```

- 改了 **Python**（server 端）→ 重建 server 镜像（`--build` 会自动处理）
- 改了 **Java / Vue / changelog**（web 端）→ 重建 web 镜像
- 只改 `data/config.yaml` → 重启 server 容器即可：`docker compose -f docker-compose-selfbuild.yml restart xiaozhi-esp32-server`

## 7. 常见问题

### 7.1 build 报 `error from sender: ... mysql/data/... permission denied`
`.dockerignore` 已排除运行时数据目录。若仍报，检查仓库里是否混入了 `main/xiaozhi-server/mysql/data/` 之类的运行时数据，删掉或加入 `.dockerignore`。

### 7.2 web 容器反复 Restarting（255）/ `exec /start.sh: no such file or directory`
CRLF 行尾问题。`Dockerfile-web` 已有 `sed -i 's/\r$//' /start.sh` 修复，**不要删这行**。如果是从 Windows 拉的代码，确保这行在。

### 7.3 `docker compose` 报 `no configuration file`
必须 `-f docker-compose-selfbuild.yml` 显式指定文件名（非默认名）。

### 7.4 server-base 拉不到 / build 卡在 FROM
默认走 NJU 镜像 `ghcr.nju.edu.cn/xinnan-tech/xiaozhi-esp32-server:server-base`。海外或换源可用：
```bash
SERVER_BASE_IMAGE=ghcr.io/xinnan-tech/xiaozhi-esp32-server:server-base \
  docker compose -f docker-compose-selfbuild.yml build
```

### 7.5 豆包缓存 403 / 不生效
- **403 AccessDenied.CacheService** → 火山账号未开通「推理(缓存)定价」，去火山方舟控制台开通（账号级操作，代码改不了）。
- 开了仍不触发 Session 缓存 → `doubao_cache.py` 首轮走 chat/completions 不存 response_id，待修（见仓库记忆）。

### 7.6 `git status` 显示 1000+ 文件 modified
**别慌，几乎全是 CRLF 行尾噪声**（仓库无 `.gitattributes`、`autocrlf` 未设）。真实改动用：
```bash
git diff -w --numstat | awk 'NF==3'   # 空行 = 纯 CRLF，不计
```
**不要** `git add -A`，会提交海量 CRLF 噪声。只 `git add` 真实改动的文件。

## 8. 端口 / 凭据速查

| 项 | 值 |
|---|---|
| 智控台 | `8002` |
